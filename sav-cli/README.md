# PalPanel sav_cli

Read-only Palworld `.sav` indexer sidecar written in Go.

This component is licensed under GPL-3.0-or-later. Release source archives
include the corresponding sav-cli source and vendored gooz source/license.

```bash
cd sav-cli
go test ./...
go run ./cmd/sav_cli inspect --file ../data/server/Pal/Saved/SaveGames/0/<world>/Level.sav
go run ./cmd/sav_cli index --save-dir ../data/server/Pal/Saved/SaveGames
go run ./cmd/sav_cli serve --host 127.0.0.1 --port 8090
go run ./cmd/sav_cli --version
```

The sidecar exposes the same HTTP contract used by PalPanel:

- `GET /health`
- `POST /index` with `{ "save_dir": "/path/to/world-or-save-root" }`

Known status:

- `PlZ1`, `PlZ2`, and `CNK` containers are decoded with zlib.
- `PlM1` is decoded with the open-source `gooz` Oodle/Kraken decompressor when cgo is enabled.
- Static `CGO_ENABLED=0` builds report `parser_incompatible` for `PlM1` instead of failing to compile.
- Normalization covers players, guilds, bases, worker Pals, item containers and slots, and map objects. Unknown properties are skipped with warnings where an optional relationship cannot be resolved.
- The parser is read-only and never writes back to `.sav`.

## Palworld 1.0 verification

- A fresh empty world generated by dedicated-server `v1.0.0.100427` has been verified through PlM/GVAS decompression, indexing, and cache generation without `parser_incompatible`.
- Identity-free synthetic regression data covers the 1.0 guild role/permission tail, character and item container slots, map-object models, character/group/base relationships, worker Pals, containers, and map entities.
- A user-provided local `v1.0.0.100427` non-empty world has been verified with one player, one guild, one base, one captured `PinkCat`, one assigned worker, 71 item containers, and a `PalBoxV2` map entity. The index completed without warnings or `parser_incompatible`.
- The local sample hashes are `2e401c71bd1767039223068b88c4d7e7ee9095541fde20fa5240a89589b958e8` (`Level.sav`), `ba66654bc1b54762756b159cd8ea39130a2cbb3d87ac6311118f67bd92ca5724` (`LevelMeta.sav`), and `6168d2e92c36e4ed4b63a3c50e96afcc7e2a9452b48eb4fdefda5662f660f33d` (player save). It contains player identity data, has no redistribution grant, and is intentionally excluded from Git.
- Set `PAL_LIVE_FIXTURE=/path/to/Level.sav` to run the gated local regression test.
- An older non-1.0 save from [cheahjs/palworld-save-tools](https://github.com/cheahjs/palworld-save-tools) was used only for local exploratory validation. Its license is MIT and its SHA-256 is `7dec28a2bdf7abcf04ca7f5bcd1c925e9992a4bb3002ef16b688e26828438faf`; the full save is not committed.
- No clearly licensed, non-empty public 1.0 save has been found, so there is still no redistributable public fixture. Non-empty compatibility is based on the gated local sample above.

When a core structure cannot be decoded, the indexer returns `parser_incompatible`. PalPanel retains the last successful index cache so offline views remain available while parser support is updated.
